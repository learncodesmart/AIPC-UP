# AIPC UP

AIPC UP is a local-first AI computer upgrade tool. It turns an ordinary Windows computer into an AI workbench that can use AI cores such as Claude, Codex, and OpenCode to help with development, file work, browser automation, desktop control, voice input, and remote collaboration.

## What It Does

AIPC UP is more than a chat assistant. With your authorization, it can enter your local computer workspace, understand projects and files, operate the browser and desktop apps, and move assigned tasks forward step by step.

Core capabilities:

- AI workbench for conversations, projects, files, plans, and execution
- Support for AI cores such as Claude, Codex, and OpenCode
- Custom model URLs and API keys
- One-click switching between connected AI cores
- File and project management with visible change tracking
- Browser automation for clicking, filling forms, screenshots, and web tasks
- Desktop control for Windows, mouse, and keyboard operations
- Voice and text task input
- Remote access from a phone or another computer through a browser
- Remote task connection through tools such as WeChat, Feishu, and Telegram

## Typical Use Cases

- Let AI read a project, inspect files, understand changes, and help implement tasks
- Ask AI to operate websites, fill forms, collect information, or complete repetitive browser steps
- Use desktop automation to control local applications under your authorization
- Continue work remotely, check task progress, or stop tasks when needed
- Manage AI coding sessions, plans, files, and execution results in one local workbench

## Quick Start

1. Download the Windows installer from [Releases](https://github.com/learncodesmart/AIPC-UP/releases/latest).
2. Install and open AIPC UP.
3. Enter your model endpoint and API key.
4. Start a conversation by text or voice.
5. Authorize browser, desktop, or remote access only when you need those capabilities.

## Safety Model

AIPC UP is designed for real computer operations, so authorization and control matter.

- Local-first: files, projects, and execution environments stay on your own computer first.
- Authorized access: browser remote access and communication connections require explicit authorization or binding.
- Visible execution: tasks can show progress and operation records.
- Task control: automation can be paused, resumed, or stopped.
- Real-world risk: browser and desktop automation can perform real actions, so confirm important operations before running them.
